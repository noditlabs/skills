# eth_getTransactionReceipt

> Returns the receipt of a transaction by transaction hash. Note: receipts are not available for pending transactions.

- **Category**: Node API (JSON-RPC)
- **Official Docs**: https://developer.nodit.io/reference/eth_getTransactionReceipt

## Supported Chains
arbitrum, arc, avalanche, base, bnb, ethereum, giwa, kaia, luniverse, optimism, polygon (common EVM method)

## Method
`eth_getTransactionReceipt`

## Parameters
| Name | Type | Required | Description |
|------|------|----------|-------------|
| `params` | `array` | Yes | Array of 1 item |
| `params[0]` - Transaction Hash | `string` | Yes | The transaction hash as a 64-character hex string (e.g., `"0xda148d856aef6d77..."`) |

> **Note**: Response time may be longer depending on the number of event logs in the transaction and the transaction's position within the block.

## Returns
A transaction receipt object, or `null` when no receipt was found. The receipt object contains:
- `blockHash` - Hash of the block containing the transaction
- `blockNumber` - Block number containing the transaction
- `contractAddress` - The contract address created if the transaction was a contract creation, otherwise `null`
- `cumulativeGasUsed` - Total amount of gas used in the block up to and including this transaction
- `effectiveGasPrice` - The actual gas price paid per unit of gas
- `from` - Address of the sender
- `gasUsed` - Amount of gas used by this specific transaction
- `logs` - Array of log objects generated by this transaction
  - `address` - Address from which this log originated
  - `topics` - Array of 0 to 4 indexed log arguments (32-byte hex strings)
  - `data` - Non-indexed arguments of the log
  - `blockNumber` - Block number
  - `transactionHash` - Transaction hash
  - `transactionIndex` - Transaction index in the block
  - `blockHash` - Block hash
  - `logIndex` - Log index in the block
  - `removed` - `true` if log was removed due to chain reorganization
- `logsBloom` - Bloom filter for light clients to quickly retrieve related logs
- `status` - `"0x1"` for success, `"0x0"` for failure
- `to` - Address of the receiver
- `transactionHash` - Hash of the transaction
- `transactionIndex` - Transaction index position in the block
- `type` - Transaction type

## Example Request
```json
{
  "id": 1,
  "jsonrpc": "2.0",
  "method": "eth_getTransactionReceipt",
  "params": [
    "0xda148d856aef6d77d0b76c90ef1091ffe77afe9ee9b1c6cc23f28f042f198bd8"
  ]
}
```
